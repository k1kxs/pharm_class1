
# Подготовка к переносу проекта на удаленный сервер

Для успешного переноса проекта на удаленный сервер с сохранением данных потребуется:

## 1. Настроить серверное хранение данных
- Настроить MongoDB (уже указана в server.ts)
- Изменить dataAPI для работы с сервером вместо localStorage
- Реализовать маршруты для CRUD-операций с классификациями в `/routes/data.ts`
- Создать схемы MongoDB для хранения структуры данных

## 2. Обновить аутентификацию
- Заменить локальную проверку пароля на серверную
- Добавить управление пользователями (админ/студенты)
- Настроить JWT аутентификацию с правильной валидацией

## 3. API для работы с данными
- Создать полноценное API для:
  - Получения классификаций (для студентов)
  - Редактирования (только для админа)
  - Экспорта в PDF (существующая функциональность)

## 4. Подготовка к деплою
- Собрать фронтенд (`npm run build`)
- Настроить статическую раздачу фронтенда через Express
- Проверить переменные окружения (порты, пути, URL)
- Настроить CORS для безопасных запросов

## 5. Оптимизация
- Добавить кеширование на сервере
- Сжатие статических ресурсов
- Настроить правильное логирование

## 6. Безопасность
- Защитить API ключами/токенами
- Включить HTTPS
- Санитизировать ввод пользователей
- Ограничить доступ к административным функциям
